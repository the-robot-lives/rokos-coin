# US-007: Unstake ROKO after the cooldown period

**As a** independent miner and compute provider,
**I want to** initiate an unstake request and receive my ROKO collateral back after the mandatory cooldown period has elapsed,
**So that** I can exit the network or reduce my stake while still giving the protocol time to resolve any outstanding disputes against my node.

## Acceptance Criteria
- [ ] Unstake request immediately sets miner status to "exiting" and stops new job assignments to that miner
- [ ] The cooldown duration (e.g., 7 days) is a governance-controlled parameter readable from the contract
- [ ] Tokens are locked for the full cooldown period; any attempt to withdraw before cooldown completes is rejected with a clear error
- [ ] After cooldown, a single withdraw call transfers the full unstaked amount minus any pending slashing deductions
- [ ] An UnstakeInitiated and UnstakeCompleted event are emitted at each phase for indexer and UI tracking

## Notes
Cooldown protects against stake-and-slash evasion. If the miner has open slashing disputes at cooldown completion, funds must remain locked until disputes resolve. Partial unstakes (reducing below minimum) should trigger the same exit flow.
