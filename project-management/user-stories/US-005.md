# US-005: Receive ROKO rewards for serving inference requests

**As a** independent miner and compute provider,
**I want to** automatically receive ROKO token rewards each time I successfully serve a verified inference request,
**So that** my hardware investment is compensated and I have ongoing economic incentive to remain an honest, available provider.

## Acceptance Criteria
- [ ] Rewards are credited to my miner account (claimable balance) upon successful proof-of-inference verification, not before
- [ ] Reward amount is calculated per the published formula (base rate × hardware tier multiplier × job complexity weight)
- [ ] A RewardAccrued event is emitted for every credited reward, linking the job ID, miner address, and ROKO amount
- [ ] Rewards accumulate in a claimable pool; actual token transfer to wallet occurs only when the miner calls the withdraw function (US-027)
- [ ] If verification fails or the result is disputed, reward is withheld and the dispute resolution path is surfaced in the UI

## Notes
Lazy payout (accumulate then withdraw) reduces gas costs. Reward rate must be sustainable relative to emission schedule. Integrate with proof-of-inference verification pipeline — reward triggers only after the verification phase passes.
