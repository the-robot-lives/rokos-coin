# US-019: Add liquidity to ROKO/ETH pool on Uniswap

**As a** DeFi investor and token holder,
**I want to** add liquidity to the ROKO/ETH pool on Uniswap and receive LP tokens representing my position,
**So that** I can earn trading fees while supporting market liquidity, which benefits the broader ecosystem.

## Acceptance Criteria
- [ ] Protocol documentation provides the official Uniswap pool address(es) and step-by-step instructions for adding liquidity
- [ ] A guided flow in the protocol UI (or deep-link to Uniswap) pre-selects the correct pool and token addresses
- [ ] LP token address is documented so users can import it into their wallets and track positions
- [ ] Protocol treasury's LP position (if any) is disclosed on the token distribution page (US-013) for transparency
- [ ] Documentation warns about impermanent loss risk, especially given ROKO's volatility as a newer token

## Notes
The protocol should not mandate or incentivize a specific AMM exclusively — document the primary pool but do not lock liquidity provision to one venue. If the protocol runs an incentivized liquidity mining program (ROKO rewards for LP tokens), that should be a separate story.
