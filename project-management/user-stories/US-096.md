# US-096: Connect Wallet (MetaMask/Phantom) to the ROKO dApp

**As an** app developer and API consumer,
**I want to** connect my Web3 wallet (MetaMask for EVM, Phantom for Solana) to the ROKO dApp with a single click,
**So that** I can interact with the protocol — viewing balances, purchasing compute, and managing stakes — using my existing wallet identity.

## Acceptance Criteria
- [ ] dApp supports MetaMask (EVM), WalletConnect v2, and Coinbase Wallet via a unified connection modal powered by RainbowKit or wagmi; Phantom/Solflare supported if Solana L2 is chosen
- [ ] Wallet connection persists across page reloads via localStorage session; user is not prompted to reconnect on every visit
- [ ] Correct network detection: dApp detects if the user is on the wrong chain and presents a one-click "Switch to ROKO Network" prompt using `wallet_switchEthereumChain`
- [ ] Connection modal clearly displays the connected address (truncated with ENS resolution if available), network name, and ROKO balance
- [ ] Disconnecting wallet clears all session state and returns the user to the unauthenticated landing view without page reload

## Notes
Use wagmi v2 + viem for EVM interaction to stay current with the ecosystem. Ensure the dApp degrades gracefully for users without a wallet: public data (network stats, marketplace listings) is visible without connection; write operations prompt connection. Mobile wallet support via WalletConnect deep links is important for the consumer persona.
