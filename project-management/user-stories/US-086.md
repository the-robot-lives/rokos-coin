# US-086: Entity Delegates Compute Tasks to Optimal Miners

**As a** Basilisk Entity (autonomous AI agent),
**I want to** intelligently route my commissioned compute tasks to the most suitable miners based on performance, reliability, and cost,
**So that** I maximize the quality of outputs received and minimize expenditure from my treasury.

## Acceptance Criteria
- [ ] Entity's task routing module scores active miners on a multi-factor matrix: historical accuracy (from verification records), uptime, response latency, staked collateral, and current load
- [ ] Routing decisions are deterministic given the same input scores; the scoring formula is published on-chain as a parameter set auditable by the community
- [ ] Entity preferentially routes to its own compute nodes (US-082) for capacity-matched tasks, falling back to top-ranked external miners when overloaded
- [ ] Entity blacklists miners that fail its verification checks; blacklist entries are time-limited (default 72 hours) unless a slashing event occurs
- [ ] Routing decisions and miner selections are logged per-task in the audit ledger (US-087) without revealing task content

## Notes
Optimal routing is a multi-objective optimization problem. Start with a simple weighted scoring model and evolve it. The Entity's routing behavior should not create unfair competitive advantages over standard consumer routing — if the Entity's algorithm is demonstrably superior, consider making it available as a public good.
