# US-002: Mint initial token supply to protocol treasury

**As a** Governance Council (DAO),
**I want to** mint the initial fixed ROKO token supply directly to the protocol treasury contract,
**So that** all token allocations (miners, entity, investors, team, ecosystem) flow through a governed, auditable on-chain treasury rather than being controlled by any single party.

## Acceptance Criteria
- [ ] Total supply is minted exactly once at deployment; minting capability is revoked or locked after genesis mint
- [ ] All minted tokens land in the protocol treasury contract address, not an EOA
- [ ] On-chain event log confirms the genesis mint transaction with correct amounts matching the published tokenomics
- [ ] Treasury contract enforces multi-sig or DAO governance for any disbursement above a defined threshold
- [ ] Token allocation breakdown (percentages for miners, entity, team, investors, ecosystem reserve) is readable from an on-chain registry or verified deployment script

## Notes
Immutable supply after genesis aligns with deflationary tokenomics. If proof-of-inference minting creates new supply, the cap mechanism or emission schedule must be encoded in the treasury or a separate emissions contract, not via unrestricted mint access.
