# US-044: Use a TypeScript SDK to Interact with the ROKO API

**As a** app developer/API consumer,
**I want to** use an official TypeScript/JavaScript SDK to interact with the ROKO compute network,
**So that** I can integrate inference into my web or Node.js application without writing low-level contract calls.

## Acceptance Criteria
- [ ] The `@roko/sdk` npm package provides typed methods: `submitInference`, `batchSubmit`, `getRequestStatus`, `fetchResult`, `estimateFee`, and `reclaimExpired`
- [ ] SDK handles wallet connection (ethers.js / viem compatible), ERC-20 approval flow, and event subscription out of the box with minimal configuration
- [ ] Full TypeScript types and JSDoc are provided for all public methods, parameters, and return types; the package ships with `.d.ts` declarations
- [ ] SDK includes a streaming result helper that polls or subscribes to `ComputeComplete` events and resolves a Promise when the result is available, with configurable timeout
- [ ] A `README.md` with quickstart (submit → wait → fetch result in under 20 lines of code) and full API reference are published alongside the package

## Notes
The SDK is the primary developer touchpoint; quality here directly affects ecosystem adoption. Consider publishing to both npm and a bundled CDN-compatible ESM build. Wallet-agnostic design (accept a `signer` interface) prevents lock-in to a single web3 library.
