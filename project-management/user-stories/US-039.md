# US-039: Reclaim Escrowed ROKO if Task Deadline Passes Without Result

**As a** app developer/API consumer,
**I want to** automatically reclaim my escrowed ROKO tokens if no result is submitted before the request deadline,
**So that** I never permanently lose funds due to a miner going offline or dropping a job.

## Acceptance Criteria
- [ ] Any request with `status == PENDING | ASSIGNED` whose `deadline` has passed can be cancelled by calling `reclaimExpired(requestId)` from the original requester address
- [ ] On successful reclaim, the full escrowed ROKO amount is returned to the requester and the request status is set to `EXPIRED`
- [ ] A miner assigned to an expired job that subsequently submits a result receives a revert with `RequestExpired` rather than a payment
- [ ] If a miner was slashed for non-delivery (per staking rules), any slash proceeds are distributed to the requester as partial compensation
- [ ] `reclaimExpired` emits a `RequestExpired` event with `requestId`, `requesterAddress`, and `reclaimedAmount`

## Notes
Deadline is set by the requester at submission time, bounded by a governance-defined min/max range (e.g., 30 seconds to 1 hour). The slashing linkage (criterion 4) depends on the miner staking subsystem defined in earlier mining stories.
