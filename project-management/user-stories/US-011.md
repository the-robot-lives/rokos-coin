# US-011: Set my hardware tier based on benchmark results

**As a** independent miner and compute provider,
**I want to** have my GPU hardware classified into a tier after running the official benchmark suite,
**So that** I receive the correct tier-adjusted rewards and am matched to job types appropriate for my actual hardware capabilities.

## Acceptance Criteria
- [ ] Benchmark suite produces a signed result (tokens/sec, VRAM, precision support) that can be submitted on-chain as a hardware tier claim
- [ ] Tier assignment (e.g., Tier 1: consumer GPU, Tier 2: prosumer, Tier 3: datacenter) is written to the miner's on-chain record upon successful benchmark submission
- [ ] Tier claims are subject to periodic re-verification; a miner cannot self-assign a higher tier than the benchmark supports
- [ ] If benchmark results are disputed or inconsistent with actual job performance, the tier can be downgraded by the verification layer
- [ ] Tier is displayed prominently on the miner dashboard (US-006) and on the public leaderboard (US-022)

## Notes
Benchmark runner should be open-source and reproducible. Results should include a hardware fingerprint to prevent replay attacks. Tier boundary thresholds are governance-controlled parameters (US-021).
