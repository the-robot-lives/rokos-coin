# US-047: Filter Miners by Hardware Tier, Latency, and Pricing

**As a** enterprise architect,
**I want to** filter the miner marketplace by hardware tier, latency SLA, and price range,
**So that** I can quickly shortlist providers that meet our performance budget without manually reviewing every listing.

## Acceptance Criteria
- [ ] Filter panel exposes: hardware tier (A100/H100/RTX 4090/etc.), maximum acceptable p95 latency (slider, ms), price range (ROKO per 1K tokens, min/max), minimum uptime percentage, and minimum reputation score
- [ ] Filters apply client-side (for cached data) with a server fallback for filtered result counts exceeding 100, ensuring sub-200ms UI response for typical queries
- [ ] Active filters persist in the URL query string so filtered views can be bookmarked and shared
- [ ] A "smart match" button auto-populates filters based on a natural-language workload description (e.g., "low-latency chat, 500 req/min, GPT-4-class model")
- [ ] Filter state displays a result count badge ("47 miners match") that updates as filters change

## Notes
The "smart match" feature (criterion 4) is an LLM-assisted UX enhancement that can be a v2 feature. Core filtering (criteria 1-3) is v1. URL persistence is critical for enterprise procurement workflows where multiple stakeholders review the same shortlist.
