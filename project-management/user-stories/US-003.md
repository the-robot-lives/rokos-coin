# US-003: Register as a compute miner with GPU capabilities

**As a** independent miner and compute provider,
**I want to** register my GPU hardware on-chain as a compute miner,
**So that** the network recognizes my node as an eligible provider and begins routing inference requests to me.

## Acceptance Criteria
- [ ] Registration transaction records my wallet address, declared hardware tier, and a public endpoint (encrypted or hashed) on-chain
- [ ] Registration requires a minimum stake deposit (see US-004) or at least prompts the user to stake before activation
- [ ] Duplicate registrations from the same wallet are rejected or update the existing record without creating duplicates
- [ ] A MinerRegistered event is emitted with all relevant fields so indexers can track the provider set
- [ ] Registration metadata (GPU model, VRAM, region) can be updated by the miner via a separate on-chain call without re-registering from scratch

## Notes
Hardware claims are self-reported at registration; verification happens asynchronously via benchmark (US-034) and TEE attestation (US-033). The registration record should store a commitment hash of hardware specs, not raw PII or exact IP addresses.
