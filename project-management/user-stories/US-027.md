# US-027: Withdraw accumulated ROKO rewards to my wallet

**As a** independent miner and compute provider,
**I want to** withdraw my accumulated ROKO reward balance from the protocol contract to my personal wallet at any time,
**So that** I can realize my earnings, manage my cash flow, and use or trade the tokens as I choose.

## Acceptance Criteria
- [ ] A withdraw function transfers the caller's entire pending reward balance (or a specified partial amount) to their wallet in a single transaction
- [ ] Withdrawal is blocked if the miner has an open slashing dispute — pending amount is displayed with a "locked pending dispute" label
- [ ] Transaction gas cost is displayed as an estimate before the user confirms, helping miners decide when to batch withdrawals
- [ ] Withdrawal event is emitted with miner address, amount, and timestamp for indexer and tax reporting purposes
- [ ] Rewards balance in the dashboard (US-006) updates to zero (or reduced) immediately after a successful withdrawal transaction

## Notes
Gas optimization: reward accumulation in contract state (lazy payout) is the right pattern — avoid per-job on-chain transfers. Consider adding a gasless withdrawal option via meta-transaction relay if L2 gas is non-trivial. Provide CSV export of withdrawal history for tax reporting.
