# US-043: Batch Multiple Inference Requests in a Single Transaction

**As a** app developer/API consumer,
**I want to** submit multiple inference requests in a single on-chain transaction,
**So that** I reduce gas costs and latency when my application needs to fan out many parallel inference calls.

## Acceptance Criteria
- [ ] A `submitBatch(requests[])` function accepts an array of request structs (each with `modelId`, `promptHash`, `maxFee`, `deadline`) and returns an array of `requestId`s
- [ ] Total ROKO required for all requests is transferred in a single ERC-20 `transferFrom` call, reducing gas overhead vs. individual submissions
- [ ] Each request in the batch is independently tracked and settled; one request failing or expiring does not affect others in the same batch
- [ ] Batch size is capped at a governance-defined maximum (e.g., 50 requests) to prevent block gas limit issues
- [ ] The SDK `batchSubmit` method handles prompt hashing, fee estimation, and single-approval flow transparently for the developer

## Notes
Batch submission is particularly valuable for embedding generation, bulk document processing, and re-ranking pipelines. The independent settlement requirement (criterion 3) means the contract must avoid any shared state between batch members after submission.
