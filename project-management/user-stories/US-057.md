# US-057: Rate Limit My API Usage to Stay Within Budget

**As a** app developer/API consumer,
**I want to** configure spending limits and request rate caps on my ROKO API account,
**So that** runaway code or unexpected traffic spikes don't drain my ROKO balance faster than I intend.

## Acceptance Criteria
- [ ] Developers can set a daily and monthly ROKO spend limit via the developer dashboard; requests that would exceed the active limit are rejected by the off-chain gateway before hitting the contract, preserving gas
- [ ] A per-minute request rate cap (e.g., max 60 req/min) can be configured per API key; requests exceeding the rate cap receive an HTTP 429 with a `Retry-After` header
- [ ] Spend limits and rate caps take effect within 60 seconds of being saved; the dashboard displays current period utilization as a progress bar with warning at 80% and 95% thresholds
- [ ] When a spend limit is hit, developers receive an email/webhook alert and can optionally configure an auto-topup from a designated ROKO balance above a minimum floor
- [ ] Rate limit configuration is per-API-key so developers can assign different budgets to dev, staging, and production environments

## Notes
Spend limits are enforced off-chain (pre-gateway) for efficiency; they are a safeguard, not a hard cryptographic constraint. The auto-topup feature (criterion 4) requires a pre-authorized on-chain allowance and should be opt-in with a configured maximum per-topup amount to prevent unlimited draining.
