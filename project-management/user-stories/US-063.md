# US-063: Access a Testnet Faucet for Development and Testing

**As a** app developer/API consumer,
**I want to** receive testnet ROKO tokens from a faucet without spending real funds,
**So that** I can develop, test, and debug my integration against a fully functional test environment before deploying to mainnet.

## Acceptance Criteria
- [ ] A public testnet faucet at `faucet.roko.network` (or equivalent) dispenses a fixed amount of testnet ROKO (e.g., 100 tROKO) per wallet address per 24-hour period via a simple web UI and REST API
- [ ] The faucet REST endpoint `POST /drip` accepts `{ address, captchaToken }` and returns `{ txHash, amount, nextDripAt }` with appropriate rate limiting per IP and per address
- [ ] A complete testnet environment mirrors all mainnet contracts with the same ABI, a test miner fleet running actual inference, and a testnet block explorer with labeled contracts
- [ ] The developer documentation landing page prominently links to the testnet faucet and includes a "5-minute quickstart" that goes from faucet claim to first successful inference result end-to-end
- [ ] Testnet transactions are clearly watermarked in the SDK and dashboard (e.g., "TESTNET" banner) to prevent accidental mainnet deployment of testnet-configured code

## Notes
A frictionless testnet experience is the single most important developer acquisition tool. The 5-minute quickstart (criterion 4) should be continuously integration-tested to ensure it never breaks. Consider a GitHub Actions-based faucet claim for CI/CD pipelines that avoids captcha.
