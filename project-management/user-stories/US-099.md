# US-099: Submit Inference Requests via a Web Form with Cost Preview

**As an** app developer and API consumer,
**I want to** submit inference requests directly through the dApp's web interface with a real-time cost estimate before confirming the transaction,
**So that** I can test model outputs interactively and understand pricing before integrating the API into my application.

## Acceptance Criteria
- [ ] Inference submission form includes: model/capability selector, input text area (or file upload for multimodal), priority tier selector (standard/express/guaranteed), and optional preferred miner address
- [ ] Cost preview updates in real time as inputs change, showing: estimated ROKO cost, USD equivalent at current market price, estimated queue wait time per tier, and selected miner reliability score
- [ ] Submitting the form constructs and sends the on-chain payment + job request transaction; user sees a confirmation modal with full transaction details before signing
- [ ] After submission, job status is tracked live on a results page: Queued → Processing → Verifying → Complete, with the output displayed inline when available
- [ ] ROKO balance check prevents submission if wallet balance is insufficient; a "Get ROKO" link to a swap interface is shown in the error state

## Notes
This web form is primarily a developer testing tool, not the production integration path (which is the REST API). Design it for rapid iteration and exploratory use. Consider a "Try without wallet" demo mode with a testnet faucet for new users. Output rendering should support markdown, code blocks, and base64 images for multimodal models.
