# US-021: Propose a change to the miner slashing threshold

**As a** Governance Council (DAO),
**I want to** submit and vote on proposals to adjust the miner slashing threshold and penalty percentages,
**So that** the network can calibrate the economic security model in response to observed attack patterns or evolving hardware and verification standards.

## Acceptance Criteria
- [ ] Slashing threshold parameters (minimum deviation triggering a slash, penalty percentage tiers) are stored as governance-controlled on-chain parameters
- [ ] A proposal to change slashing parameters follows the same governance flow as other proposals (submit, vote, timelock, execute)
- [ ] Parameter change is applied only to jobs submitted after the timelock execution block — no retroactive slashing under new rules
- [ ] The current slashing parameter values are displayed prominently in the miner onboarding flow and dashboard so miners always know the rules they are operating under
- [ ] Audit trail of all slashing parameter changes is accessible with proposal IDs, vote outcomes, and effective dates

## Notes
Slashing parameter governance is a high-stakes operation. Consider requiring a security auditor sign-off (multi-sig co-signer) for slash threshold reductions below a safety floor to prevent governance attacks that weaken the security model.
