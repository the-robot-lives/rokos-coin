# US-037: Pay for Inference with ETH/USDC (Auto-Swapped to ROKO)

**As a** app developer/API consumer,
**I want to** pay for inference using ETH or USDC without manually acquiring ROKO first,
**So that** I can integrate the API without requiring users to manage a separate token.

## Acceptance Criteria
- [ ] The API gateway contract accepts ETH or whitelisted stablecoins and performs a DEX swap (e.g., Uniswap V3) to ROKO atomically within the same transaction
- [ ] Developer receives a quote for the ROKO equivalent amount before submitting, with configurable slippage tolerance (default 0.5%)
- [ ] If the swap fails or slippage exceeds tolerance, the entire transaction reverts and no funds are moved
- [ ] The settled `requestId` records the original payment token and amount alongside the ROKO equivalent for receipt/audit purposes
- [ ] A `PaymentConverted` event is emitted with `inputToken`, `inputAmount`, `rokoAmount`, and `requestId`

## Notes
This is a UX bridge to reduce onboarding friction. The protocol is ROKO-native; the swap wrapper is a convenience layer. Governance may whitelist/blacklist accepted input tokens. Slippage protection is critical to prevent sandwich attacks on payment.
