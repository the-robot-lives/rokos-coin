# US-064: View API Documentation with Interactive Examples

**As a** app developer/API consumer,
**I want to** access comprehensive API documentation with runnable interactive examples,
**So that** I can understand the protocol quickly, experiment with calls in-browser, and copy working code into my project without trial and error.

## Acceptance Criteria
- [ ] API reference documentation covers all contract functions, SDK methods, event schemas, error codes, and REST endpoints with parameter descriptions, types, and return value documentation
- [ ] Every documented method includes at least one interactive code example runnable in-browser (connected to testnet) via an embedded playground, with output displayed inline
- [ ] Documentation includes conceptual guides: "How inference routing works," "Escrow and settlement lifecycle," "Handling failures and retries," and "Security model" — each with diagrams
- [ ] A search function with keyword and tag filtering returns relevant docs, guides, and examples in under 200ms
- [ ] Documentation is versioned and clearly marks which SDK version each example targets; a version selector allows switching between docs for different protocol versions

## Notes
Documentation quality is a leading indicator of ecosystem adoption. The interactive playground (criterion 2) dramatically reduces time-to-first-success. Consider using an OpenAPI spec as the source of truth for REST endpoint docs to keep docs and implementation in sync. Diagrams (criterion 3) should be generated from code (e.g., Mermaid) rather than hand-crafted images to stay current.
