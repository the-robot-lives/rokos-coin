# US-045: Use a Python SDK to Interact with the ROKO API

**As a** AI researcher,
**I want to** use an official Python SDK to submit inference and training jobs to the ROKO network,
**So that** I can integrate decentralized compute into my research notebooks and ML pipelines without learning Web3 tooling.

## Acceptance Criteria
- [ ] The `roko-sdk` PyPI package provides a `RokoClient` class with methods: `submit_inference`, `batch_submit`, `get_status`, `fetch_result`, `estimate_fee`, and `reclaim_expired`
- [ ] SDK supports both sync and async (`asyncio`) usage patterns, with the async variant compatible with Jupyter notebooks (`await`-friendly)
- [ ] Private key / wallet management supports: raw private key (env var), hardware wallet via `eth_account`, and read-only mode for status/result queries
- [ ] A `pandas`-compatible `ResultDataFrame` helper converts batch inference results into a structured DataFrame for downstream analysis
- [ ] Documentation includes a Jupyter notebook example demonstrating: wallet setup, fee estimation, batch embedding generation, and result collection

## Notes
Researchers are the primary driver of training job demand and academic use cases. Python SDK parity with the TypeScript SDK is critical. Consider a `langchain` integration as a community extension. The `ResultDataFrame` helper (criterion 4) is a researcher-specific quality-of-life feature not needed in the TS SDK.
