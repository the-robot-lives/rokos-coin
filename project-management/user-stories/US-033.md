# US-033: Verify miner hardware attestation via TEE

**As a** security auditor,
**I want to** verify that participating miners are using Trusted Execution Environment (TEE) attestation to cryptographically prove their hardware configuration,
**So that** inference results can be attributed to verified hardware and the verification pipeline has a high-integrity anchor beyond self-reported claims.

## Acceptance Criteria
- [ ] Miners with TEE-capable hardware (Intel TDX, AMD SEV-SNP, or equivalent) can generate and submit an attestation quote to the verification contract
- [ ] Attestation quote is verified on-chain or via a trusted off-chain verifier with the result recorded in the miner's on-chain record
- [ ] TEE-attested miners receive a "verified hardware" badge visible on the leaderboard (US-022) and a reward multiplier for TEE-attested inference jobs
- [ ] Attestation quotes include a freshness nonce tied to the current epoch to prevent replay of old attestations
- [ ] Attestation failure or expiry without renewal downgrades the miner from "TEE-verified" to "standard" tier without interrupting service

## Notes
TEE attestation is phase three of the four-phase verification pipeline (redundant execution → statistical sampling → TEE → ZK proofs). Not all miners will have TEE hardware; it should be an optional upgrade path, not a prerequisite. On-chain quote verification is gas-intensive — consider off-chain verifier with optimistic challenge period.
