# US-034: Run benchmark suite to prove hardware capabilities

**As a** independent miner and compute provider,
**I want to** run the official ROKO benchmark suite on my hardware and submit the signed results on-chain,
**So that** my hardware tier is accurately assigned, my rewards are correctly calibrated, and the network can trust that my declared capabilities are real.

## Acceptance Criteria
- [ ] Benchmark suite is open-source, versioned, and distributed via the official protocol repo or package registry
- [ ] Benchmark measures relevant metrics: tokens/sec for standard model sizes, VRAM capacity, precision support (FP32/FP16/INT8), and latency percentiles
- [ ] Results are signed by the benchmark runner with a hardware fingerprint (CPU/GPU serial hash) to prevent result fabrication or replay
- [ ] Signed results can be submitted in a single on-chain transaction that updates the miner's tier record if the new results meet a different tier threshold
- [ ] Benchmark can be re-run and re-submitted at any time; the network may also request a re-benchmark via a challenge mechanism if live performance diverges significantly from declared tier

## Notes
The benchmark runner should be deterministic and produce consistent results across runs on the same hardware to within a defined tolerance. Consider a reference benchmark baseline (run on known hardware by the protocol team) that miners can compare against. Integrate with TEE attestation (US-033) for highest assurance tier: TEE-signed benchmark results.
