# US-100: View Network Health Dashboard with Miner Count, Throughput, and Uptime

**As a** Governance Council member,
**I want to** monitor a real-time network health dashboard showing aggregate miner count, compute throughput, request latency, uptime, and economic metrics,
**So that** the DAO can make informed governance decisions about network capacity, incentive parameters, and expansion priorities.

## Acceptance Criteria
- [ ] Dashboard displays real-time metrics: active miner count (total and by capability category), requests per minute (current and 24h average), median and p99 response latency, network uptime percentage (rolling 30-day), total ROKO staked, and current epoch reward rate
- [ ] Historical charts for all key metrics are available with 1h, 24h, 7d, and 30d time windows; charts are zoomable and include anomaly annotations
- [ ] Entity treasury section shows current balance, 30-day inflow/outflow, and a mini balance history chart
- [ ] Geographic distribution map (best-effort, based on voluntary miner disclosures) shows node density by region
- [ ] Dashboard is publicly accessible without wallet connection; data refreshes every 30 seconds via WebSocket or polling; a data freshness indicator shows the last update timestamp

## Notes
This dashboard serves as the protocol's public health signal — it should be embeddable and shareable. Consider a dedicated status page subdomain (status.roko.network). Alert thresholds for governance notifications: active miners drop below 50, throughput drops 30% vs. 7-day average, any miner concentration exceeding 15% of total capacity. Open Metrics / Prometheus export endpoint for enterprise integrations.
