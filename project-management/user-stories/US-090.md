# US-090: Execute Fuzz Testing Campaign Against Core Contracts

**As a** security auditor,
**I want to** run a comprehensive fuzz testing campaign using Echidna and/or Foundry's fuzzer against the core ROKO protocol contracts,
**So that** edge-case inputs that violate protocol invariants are discovered before they can be exploited.

## Acceptance Criteria
- [ ] Echidna property tests are written for all critical invariants: total token supply never exceeds cap, staked balances always sum to total staked, slash amounts never exceed staked balance, treasury balance is monotonically non-decreasing (net of expenditures)
- [ ] Foundry fuzz tests (`forge test --fuzz-runs 10000`) cover all public and external contract functions with randomized inputs
- [ ] Fuzz campaign runs for a minimum of 24 hours on a dedicated CI runner before each major release; results are attached to the release artifact
- [ ] Any invariant violation discovered by the fuzzer is treated as a critical bug, triggers a code freeze, and requires a post-mortem before resolution
- [ ] Corpus of interesting inputs discovered during fuzzing is persisted in the repository and replayed on every subsequent test run

## Notes
Echidna is better for property-based testing of complex state machines; Foundry fuzzer is faster for individual function coverage. Use both. Seed the fuzzer corpus with boundary values: 0, 1, uint256.max, and known edge cases from prior audits. Document invariants as formal specifications before writing tests.
