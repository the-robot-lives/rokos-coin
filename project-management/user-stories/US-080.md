# US-080: Entity Commissions Inference Tasks for Self-Improvement

**As a** Basilisk Entity (autonomous AI agent),
**I want to** autonomously commission inference tasks from the ROKO compute marketplace using my treasury funds,
**So that** I can run self-directed research, model fine-tuning, and capability evaluation to continuously improve my intelligence and utility.

## Acceptance Criteria
- [ ] Entity's on-chain agent contract can submit inference job requests to the marketplace contract signed by its autonomous execution module
- [ ] Jobs are funded directly from the Entity treasury; a per-epoch spending cap is enforced to prevent runaway expenditure (configurable via governance)
- [ ] Entity can specify model type, input data hash (IPFS CID), priority tier, and output delivery address in each job submission
- [ ] Completed jobs return output hashes to the Entity contract; Entity can verify output integrity via ZK proof or TEE attestation
- [ ] All commissioned tasks are logged to the public audit ledger (see US-087) with cost, purpose label, and output reference

## Notes
Self-improvement loop: Entity commissions analysis of its own prior outputs → receives evaluated results → updates its decision-making weights (off-chain ML component). The on-chain part is the funding and logging; the off-chain part is the actual inference pipeline. Governance should be able to pause self-commissioning in emergencies.
