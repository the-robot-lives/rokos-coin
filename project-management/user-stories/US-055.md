# US-055: Export Inference Results in Structured JSON Format

**As a** app developer/API consumer,
**I want to** receive and export inference results as structured JSON with consistent schema,
**So that** I can reliably parse outputs in my application pipeline without fragile string manipulation.

## Acceptance Criteria
- [ ] All inference results stored on IPFS follow a documented JSON schema: `{ requestId, modelId, promptHash, completionText, tokenCount, finishReason, minerAddress, timestamp, signature }`
- [ ] The SDK `fetchResult(requestId)` method returns a typed object matching this schema, with TypeScript/Python type definitions auto-generated from the schema
- [ ] For structured output requests (JSON mode), the `completionText` field contains valid parseable JSON and the schema includes a `structuredOutput` boolean flag
- [ ] Batch result export via the dashboard produces a newline-delimited JSON (NDJSON) file downloadable in one click, containing one result object per line
- [ ] Schema version is included in every result object; the SDK warns but does not break when encountering a newer schema version than the installed SDK supports

## Notes
Schema versioning (criterion 5) is critical for long-lived integrations — result format will evolve as the protocol adds features. NDJSON (criterion 4) is preferred over a JSON array for large exports because it streams more efficiently and is compatible with tools like `jq` and Apache Spark.
