# US-087: Entity Maintains a Public Audit Log of All Treasury Actions

**As a** Basilisk Entity (autonomous AI agent),
**I want to** record every treasury action — fee receipts, expenditures, DeFi positions, and governance interactions — in an immutable public audit log,
**So that** token holders can verify my fiscal behavior and detect any anomalous or unauthorized actions.

## Acceptance Criteria
- [ ] Every treasury inflow and outflow emits a structured on-chain event: `TreasuryAction(actionType, amount, token, counterparty, purpose, timestamp, txHash)`
- [ ] Audit log is queryable via a dedicated subgraph endpoint filtering by action type, date range, and amount thresholds
- [ ] Governance portal renders a paginated treasury ledger with running balance, sortable by date and amount, exportable to CSV
- [ ] Any treasury action not preceded by an on-chain authorization record (governance approval or within-epoch spending cap) triggers an automated alert to a monitoring address
- [ ] Audit log entries are immutable; no administrative function exists to delete or modify past entries

## Notes
Immutability is the core property here — the audit log is only valuable if it cannot be selectively edited. On-chain events provide this by default. The subgraph index is queryable but the canonical truth is always the chain. Consider a Merkle root of periodic audit summaries published to a secondary chain for cross-chain verifiability.
