# US-036: Pay for Inference with ROKO Tokens via Escrow

**As a** app developer/API consumer,
**I want to** pay for inference by escrowing ROKO tokens in the API gateway contract,
**So that** payment is atomic with job submission and miners are guaranteed to receive funds upon valid completion.

## Acceptance Criteria
- [ ] Calling `submitRequest` transfers the specified ROKO fee from the caller's wallet into the contract escrow in the same transaction
- [ ] Escrowed funds are locked per `requestId` and cannot be withdrawn by anyone while the job is in `ASSIGNED` or `PENDING` state
- [ ] On job completion and verification, the contract releases the miner's share and protocol fee (5%) without requiring any additional caller action
- [ ] If ROKO allowance is insufficient, the transaction reverts with `ERC20InsufficientAllowance` before any state changes occur
- [ ] Escrow balance for each `requestId` is queryable via a public view function

## Notes
The escrow mechanism is the trust anchor for the entire marketplace. Miners must not be able to claim payment before a result hash is submitted, and consumers must not be able to reclaim funds during active assignment. See US-039 for the timeout/reclaim flow.
