# US-001: Deploy ERC-20 ROKO token contract on L2 testnet

**As a** independent miner and compute provider,
**I want to** deploy and interact with the ROKO ERC-20 token contract on the L2 testnet,
**So that** I can verify the token mechanics, test staking flows, and confirm the contract behaves correctly before mainnet launch.

## Acceptance Criteria
- [ ] ROKO ERC-20 contract is deployed to the target L2 testnet (e.g., Arbitrum Sepolia or Base Sepolia) with correct name, symbol, and decimals
- [ ] Contract source code is verified and published on the L2 block explorer
- [ ] Token transfer, approve, and transferFrom functions operate correctly per ERC-20 spec
- [ ] Contract owner/deployer address is set to the protocol multisig, not an individual EOA
- [ ] Testnet faucet or mint function is available for test participants to acquire ROKO for integration testing

## Notes
Contract should emit standard ERC-20 events (Transfer, Approval). Consider using OpenZeppelin ERC20 base with extensions for pausability and access control. Deployment scripts should be idempotent and committed to the repo with deterministic addresses via CREATE2 if possible.
