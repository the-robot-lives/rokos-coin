# US-041: View My API Usage History and Spend Dashboard

**As a** app developer/API consumer,
**I want to** view a dashboard showing my historical inference requests, statuses, and cumulative ROKO spend,
**So that** I can monitor costs, debug failed requests, and forecast my token budget.

## Acceptance Criteria
- [ ] Dashboard displays a paginated table of all `requestId`s associated with the connected wallet, including: submission time, model, status, fee paid, and result CID (if complete)
- [ ] Aggregate metrics shown at the top: total requests, total ROKO spent, success rate, average latency, and current month spend vs. prior month
- [ ] Each row is expandable to show the full request lifecycle timeline (submitted, assigned, completed/expired/challenged) with block timestamps
- [ ] Filters available: date range, model, status (complete/expired/challenged/pending), and fee range
- [ ] Dashboard data is sourced from indexed on-chain events (subgraph or equivalent) and refreshes within 60 seconds of new activity

## Notes
This is a developer-facing tool; UX should prioritize information density over simplicity. Consider CSV export for billing integration. The subgraph indexer must handle `ComputeRequest`, `ComputeComplete`, `RequestExpired`, and payment events.
