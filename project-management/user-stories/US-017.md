# US-017: Stake ROKO for passive compute credits (stake-for-access)

**As a** DeFi investor and token holder,
**I want to** stake my ROKO tokens and receive a continuous stream of compute credits without burning my principal,
**So that** I can use network compute capacity as a staking yield while retaining my token position.

## Acceptance Criteria
- [ ] Staking ROKO in the access staking contract generates compute credits at a published accrual rate per staked token per epoch
- [ ] Accrued credits are claimable at any time and usable for inference requests via the API gateway
- [ ] Access stake is separate from miner collateral stake — token holders who are not miners can participate
- [ ] Unstaking initiates a cooldown period (same or shorter than miner cooldown) before principal is returned
- [ ] The accrual rate and cooldown are governance-controlled parameters visible in the staking UI

## Notes
Stake-for-access model incentivizes long-term holding and reduces sell pressure. The compute credit yield must be calibrated so it doesn't cannibalize burn demand from direct API consumers. Consider a cap on non-miner staking pool size to protect network economics.
