# US-025: Set custom pricing for my specialized compute capabilities

**As a** independent miner and compute provider,
**I want to** advertise custom pricing for specialized capabilities I offer (e.g., specific model fine-tuning, rare GPU types, private inference),
**So that** I can capture premium pricing for differentiated services rather than competing only on commodity inference rates.

## Acceptance Criteria
- [ ] Miners can register one or more "capability listings" with a capability type, description, pricing in ROKO per unit, and availability
- [ ] Custom listings are visible to API consumers browsing the compute marketplace
- [ ] Minimum floor price is enforced by the contract to prevent race-to-the-bottom pricing that undermines network economics
- [ ] Consumers can select a specific miner's listing for a job, bypassing the default routing algorithm
- [ ] Listings can be updated or deactivated by the miner at any time; active jobs already matched to the listing complete at the original price

## Notes
This is the foundation of a marketplace layer on top of the base compute routing. Pricing discovery and reputation (US-022 leaderboard) together enable a functioning market. Consider escrow mechanics — consumer locks ROKO when selecting a listing, released to miner on job completion.
