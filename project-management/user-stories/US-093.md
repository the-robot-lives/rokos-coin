# US-093: Verify Oracle Price Feeds Cannot Be Manipulated for ROKO

**As a** security auditor,
**I want to** verify that all ROKO protocol components that consume price oracles (fee calculation, DeFi integrations, collateral valuation) are resistant to oracle price manipulation,
**So that** attackers cannot profit by briefly manipulating the ROKO price to extract value from the protocol.

## Acceptance Criteria
- [ ] All oracle integrations use time-weighted average prices (TWAP) with a minimum observation window of 30 minutes, not spot prices
- [ ] Oracle sources are aggregated from at least 3 independent price feeds (e.g., Chainlink, Uniswap TWAP, Band Protocol); protocol uses median value to resist individual feed manipulation
- [ ] A price circuit breaker halts ROKO-denominated operations if the reported price deviates more than 20% from the previous epoch's average; circuit breaker reset requires governance action
- [ ] Audit includes a simulation of a flash loan attack that temporarily moves ROKO price 50% and verifies no protocol function can be profitably exploited during the manipulation window
- [ ] Oracle contract addresses are stored in a governance-controlled registry; oracle updates require a 48-hour timelock to prevent sudden feed substitution

## Notes
TWAP manipulation requires sustained capital over the observation window, making it significantly more expensive than spot price manipulation. The circuit breaker is a second line of defense. Chainlink as primary oracle is preferred for mainnet; document fallback behavior if Chainlink reports stale data (heartbeat > 1 hour).
