# US-024: Monitor my miner node health and auto-restart on failure

**As a** independent miner and compute provider,
**I want to** run a monitoring agent alongside my miner software that detects failures and automatically restarts the node process,
**So that** my uptime remains high, I don't miss heartbeat windows that would reduce my availability rewards, and I avoid slashing from missed jobs.

## Acceptance Criteria
- [ ] Miner software includes or integrates with a watchdog process that monitors node health via a local health endpoint
- [ ] On process failure, crash, or health check timeout, the watchdog restarts the miner process within a configurable grace period (e.g., 30 seconds)
- [ ] Restart events are logged locally with timestamp, failure reason, and restart count
- [ ] If the node fails to recover after N restart attempts, an alert is triggered via configurable channels (email, webhook, Telegram)
- [ ] Dashboard (US-006) reflects node status in near real-time — "online," "degraded," "offline" — based on heartbeat freshness

## Notes
Auto-restart protects against transient failures but should not mask persistent errors. Implement exponential backoff after repeated failures to avoid a crash-restart loop that burns GPU resources. Node operators in datacenter environments may prefer integration with systemd or Docker restart policies.
