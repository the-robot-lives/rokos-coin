# US-067: Receive SLA Violation Refunds Automatically via Smart Contract

**As a** enterprise architect,
**I want to** receive automatic refunds when a miner violates an SLA commitment without having to manually file a dispute,
**So that** SLA enforcement is trustless and the overhead of monitoring and claiming violations doesn't fall on my operations team.

## Acceptance Criteria
- [ ] SLA violation detection runs on-chain via oracle attestation: a committee of verifier nodes samples request latency and completion rates, submits signed performance reports, and the `ReservationAgreement` contract aggregates reports to compute violation severity
- [ ] Violation severity is calculated as the percentage by which actual performance fell short of the SLA commitment over a measurement period; the refund amount is proportional to severity (e.g., 10% shortfall → 10% period fee refunded)
- [ ] Qualifying refunds are credited to the enterprise's prepaid account or wallet automatically at the end of each measurement period without requiring any consumer action
- [ ] A violation history log is emitted as on-chain events (`SLAViolationRecorded`) and displayed in the enterprise dashboard with per-period severity scores, refund amounts, and performance charts
- [ ] Miners can dispute oracle-reported violations within a governance-defined window by providing counter-evidence; confirmed false reports result in penalties against the attesting verifier nodes

## Notes
Automatic refunds without consumer action (criterion 3) is the core value proposition of this story. The oracle committee design (criterion 1) is critical — a single oracle is a centralization risk and manipulation target. Consider a minimum committee size of 5 and a supermajority threshold (e.g., 4/5 agreement) for violation confirmation. The miner dispute mechanism (criterion 5) ensures the oracle committee is itself accountable.
