# US-016: Burn ROKO to request compute (burn-and-mint model)

**As a** app developer and API consumer,
**I want to** burn ROKO tokens in exchange for verified compute credits that allow me to submit inference requests,
**So that** my application has a predictable, token-denominated access model and token burning creates deflationary pressure that benefits all holders.

## Acceptance Criteria
- [ ] Calling the burn-for-compute function burns the specified ROKO amount and mints an equivalent compute credit balance for the caller's address
- [ ] Burn-to-credit exchange rate is readable from a contract parameter and adjustable by governance
- [ ] Burned tokens are permanently removed from supply; total supply decreases and a Burn event is emitted
- [ ] Compute credits are non-transferable and tracked per address in the API gateway contract
- [ ] If the requested inference job fails network-side (not consumer error), a partial credit refund is issued per a defined refund policy

## Notes
Burn-and-mint creates a direct link between compute demand and token value. Exchange rate should reflect true compute cost to prevent arbitrage-driven over-burning. Credits should have an expiry to prevent indefinite hoarding that masks real demand.
