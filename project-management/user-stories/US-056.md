# US-056: Set Up Webhook Notifications for Completed Inference Tasks

**As a** app developer/API consumer,
**I want to** register a webhook URL that receives a push notification when each inference task completes,
**So that** my application can react to results asynchronously without polling the contract or holding open WebSocket connections.

## Acceptance Criteria
- [ ] Developers register webhook endpoints via a signed off-chain API call (authenticated by wallet signature) associated with their address or a specific API key
- [ ] The webhook service delivers a POST request within 10 seconds of a `ComputeComplete` event being indexed, with payload: `{ requestId, resultCid, status, tokenCount, settledFee, timestamp }`
- [ ] Webhook delivery includes an HMAC-SHA256 signature header (`X-Roko-Signature`) computed with a developer-provided secret so the receiver can verify authenticity
- [ ] Failed deliveries (non-2xx response or timeout) are retried with exponential backoff up to 5 attempts over 1 hour; final failure is logged and visible in the developer dashboard
- [ ] Developers can register up to 5 webhook endpoints per address, test them with synthetic payloads from the dashboard, and view per-endpoint delivery logs

## Notes
Webhooks eliminate the need for consumers to run persistent blockchain listeners, significantly reducing integration complexity. The HMAC signing (criterion 3) is non-negotiable for production use. The delivery log (criterion 5) is essential for debugging missed notifications.
