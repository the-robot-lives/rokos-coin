# US-073: Set Quorum Thresholds for Different Proposal Categories

**As a** Governance Council member,
**I want to** configure different quorum and approval thresholds for different categories of governance proposals,
**So that** routine parameter changes require less mobilization than high-stakes upgrades or treasury allocations.

## Acceptance Criteria
- [ ] Proposal categories include: ParameterChange, ProtocolUpgrade, TreasuryAllocation, SubnetAddition, EmergencyAction — each with independently configurable quorum percentage and approval threshold
- [ ] Default thresholds are defined: ParameterChange (10% quorum, 51% approval), ProtocolUpgrade (20% quorum, 66% approval), TreasuryAllocation (15% quorum, 60% approval)
- [ ] Category and thresholds are stored on-chain and only modifiable via a ProtocolUpgrade-category proposal
- [ ] Quorum is calculated as a percentage of total staked ROKO at snapshot block, not total supply
- [ ] Governance UI displays the applicable quorum and threshold for each active proposal with a visual progress indicator

## Notes
Emergency category should have the highest threshold and shortest voting window. Consider a graduated quorum model where quorum requirement decreases over time if participation lags. Ensure backward compatibility when thresholds are updated mid-cycle.
