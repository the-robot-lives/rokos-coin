# US-083: Entity Publishes Capability Reports to Token Holders

**As a** Basilisk Entity (autonomous AI agent),
**I want to** publish periodic capability and activity reports accessible to all ROKO token holders,
**So that** the community can verify my utility, monitor my growth, and maintain informed oversight of my activities.

## Acceptance Criteria
- [ ] Entity publishes a capability report on-chain (as IPFS CID) at a minimum cadence of once per governance epoch (e.g., monthly)
- [ ] Report includes: treasury balance and composition, compute tasks commissioned (count, cost, purpose categories), DeFi positions (if any), inference endpoints served, and a self-assessed capability delta
- [ ] Report submission emits a `CapabilityReportPublished(uint256 reportId, bytes32 ipfsCid, uint256 epoch)` event
- [ ] Governance portal features a "Entity Reports" section with rendered report content, historical archive, and RSS/webhook subscription for new publications
- [ ] Token holders can submit on-chain commentary (flagged questions or commendations) attached to each report, creating a public accountability thread

## Notes
Reports should be machine-readable (JSON schema) as well as human-readable (markdown). Consider a credibility score derived from verifiable on-chain metrics vs. self-reported claims. This is a key trust mechanism — reports that cannot be verified on-chain should be clearly labeled as unverified.
