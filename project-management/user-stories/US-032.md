# US-032: Bridge ROKO tokens between L1 and L2

**As a** DeFi investor and token holder,
**I want to** bridge my ROKO tokens between Ethereum mainnet (L1) and the L2 network where the protocol operates,
**So that** I can access native L2 liquidity, participate in protocol staking, and move tokens between ecosystems as market conditions warrant.

## Acceptance Criteria
- [ ] Official bridge UI supports ROKO transfers in both directions (L1 → L2 and L2 → L1) with estimated fee and time displayed before confirmation
- [ ] L2 → L1 withdrawals display the full challenge/finalization period duration (e.g., 7 days for Optimism) prominently before the user commits
- [ ] Bridge contract is audited and the canonical bridge address is displayed clearly to prevent phishing via counterfeit bridge UIs
- [ ] Bridged ROKO on L2 is 1:1 backed by locked ROKO on L1 — total supply across all chains is verifiable and displayed on the token stats page (US-010)
- [ ] Bridge transaction history (pending, completed, failed) is accessible in the user's wallet dashboard with block explorer links for each direction

## Notes
Only support bridging via the official L2's canonical bridge or a trusted, audited third-party bridge (e.g., Stargate, Hop) — document exactly which bridges are endorsed. Warn users against unofficial bridges. If ROKO is native to L2, the L1 representation is a wrapped/locked form; make this distinction clear in UI copy.
