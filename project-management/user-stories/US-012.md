# US-012: Receive tier-adjusted rewards based on hardware class

**As a** independent miner and compute provider,
**I want to** receive reward multipliers that reflect my hardware tier,
**So that** miners who invest in higher-performance hardware are appropriately compensated relative to the compute value they contribute.

## Acceptance Criteria
- [ ] Reward calculation on-chain reads the miner's current tier from the registry and applies the corresponding multiplier
- [ ] Tier multipliers are published as governance-controlled contract parameters visible to all participants
- [ ] A miner whose tier is downgraded (due to benchmark failure or verification dispute) immediately receives lower multipliers on future jobs — not retroactively
- [ ] Reward breakdown in the miner dashboard (US-006) shows base rate, tier multiplier, and final amount per job
- [ ] Tier multipliers are logged in each RewardAccrued event for auditability

## Notes
Multiplier values should balance incentivizing premium hardware without making lower tiers economically unviable. Consider a floor reward to keep consumer GPU miners participating — they provide decentralization value even if throughput is lower.
