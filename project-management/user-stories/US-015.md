# US-015: Approve token spending for the API gateway contract

**As a** app developer and API consumer,
**I want to** approve the API gateway contract to spend ROKO on my behalf up to a defined allowance,
**So that** my application can submit inference requests without requiring a manual approval transaction for every single call.

## Acceptance Criteria
- [ ] Standard ERC-20 approve function sets an allowance for the API gateway contract address
- [ ] The approval UI surfaces the exact contract address being approved with a link to the block explorer for verification
- [ ] Infinite approval and custom amount approval are both supported; infinite approval displays a clear warning about the risk
- [ ] Permit (EIP-2612) is supported so users can approve and transact in a single meta-transaction, reducing UX friction
- [ ] Current allowance for the API gateway is displayed in the developer dashboard so users know their remaining approved budget

## Notes
EIP-2612 permit support is strongly recommended to avoid double-transaction UX. The API gateway should validate that sufficient allowance exists before attempting a transferFrom, and surface a helpful error pointing to the approval flow if not.
