# US-035: Submit an Inference Request via the API Gateway Contract

**As a** app developer/API consumer,
**I want to** submit an inference request through the API gateway smart contract,
**So that** I can trigger on-chain compute task assignment and receive a deterministic job ID I can track.

## Acceptance Criteria
- [ ] Developer can call `submitRequest(modelId, promptHash, maxFee, deadline)` on the API gateway contract and receive a unique `requestId`
- [ ] Contract emits a `ComputeRequest` event containing `requestId`, `modelId`, `promptHash`, `maxFee`, `deadline`, and caller address
- [ ] Off-chain orchestrator picks up the `ComputeRequest` event and routes the job to an eligible miner within 30 seconds
- [ ] Request status transitions are trackable on-chain: `PENDING → ASSIGNED → COMPLETE | FAILED | EXPIRED`
- [ ] Submitting a request with a `maxFee` below the current floor price reverts with a descriptive error message

## Notes
The `promptHash` is a keccak256 of the actual prompt payload stored off-chain; this keeps raw data private while keeping the contract lightweight. The `modelId` maps to a registry of supported model identifiers maintained by governance.
