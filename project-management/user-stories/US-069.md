# US-069: Vote on Governance Proposals Weighted by Staked ROKO

**As a** DeFi investor and token holder,
**I want to** cast my vote on active governance proposals with weight proportional to my staked ROKO balance,
**So that** my economic stake in the protocol translates directly into decision-making influence.

## Acceptance Criteria
- [ ] Vote weight is determined by staked ROKO balance at the proposal's snapshot block, not current balance, preventing post-proposal stake manipulation
- [ ] Voter can cast For, Against, or Abstain with an optional plaintext reason stored on-chain or via IPFS hash
- [ ] Each address can vote at most once per proposal; re-voting is rejected with a clear error
- [ ] Votes are recorded on-chain and queryable by proposal ID and voter address
- [ ] dApp displays real-time vote tallies (For/Against/Abstain totals and percentages) updating as votes are cast

## Notes
Abstain votes count toward quorum but not toward For/Against outcome. Vote reason should be optional and stored as IPFS CID to minimize on-chain calldata cost. Ensure the snapshot block is set at proposal creation time.
