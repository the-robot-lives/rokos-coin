# US-078: Propose New Subnet/Capability Category for the Network

**As a** Governance Council member,
**I want to** submit a governance proposal to add a new subnet or AI capability category to the ROKO compute marketplace,
**So that** the network can expand its supported workload types in a governed, orderly fashion rather than through ad-hoc additions.

## Acceptance Criteria
- [ ] Subnet proposal includes: capability identifier, hardware requirements (GPU VRAM, driver version), benchmarking methodology, minimum stake for miners on this subnet, and a reference implementation
- [ ] Proposal passes SubnetAddition category vote (15% quorum, 55% approval) before the capability is registered on-chain
- [ ] Approved subnet is added to the on-chain capability registry with a unique subnet ID, activating routing and staking for that category
- [ ] Miners can opt into the new subnet post-activation by meeting the stake and hardware requirements defined in the proposal
- [ ] Subnet removal is also governable: a separate proposal can deprecate a subnet with a 90-day wind-down period to protect miners with active stakes

## Notes
Subnet governance allows the network to support new model types (e.g., video generation, audio synthesis) without contract upgrades. The reference implementation should include a validation harness to prevent false capability claims. Consider incentives for early miners on new subnets to bootstrap supply.
