# US-014: Transfer ROKO tokens between wallets

**As a** DeFi investor and token holder,
**I want to** transfer ROKO tokens from my wallet to any other address,
**So that** I can participate in OTC trades, move tokens between my own wallets, or send tokens to protocol contracts.

## Acceptance Criteria
- [ ] Standard ERC-20 transfer function works correctly and updates balances atomically
- [ ] Transfer to the zero address is rejected to prevent accidental burns (use explicit burn function instead)
- [ ] Transfer event is emitted with correct from, to, and value fields
- [ ] Transfer of an amount exceeding the sender's balance fails with a clear revert reason
- [ ] The protocol frontend provides a basic send interface with address validation, ENS resolution, and confirmation step before broadcasting

## Notes
Standard ERC-20 behavior — no custom logic should be added to transfer that introduces fees or restrictions beyond what is required for compliance or security. If transfer restrictions are needed (e.g., vesting cliff), implement via a separate wrapper, not by overriding base transfer.
