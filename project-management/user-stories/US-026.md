# US-026: View network-wide compute utilization metrics

**As a** Governance Council (DAO),
**I want to** view aggregate compute utilization metrics across the entire network,
**So that** governance can make informed decisions about emission rates, capacity incentives, and network health based on real supply-demand data.

## Acceptance Criteria
- [ ] A public network stats page displays: total active miners, total staked ROKO, aggregate compute capacity (TFLOPS or tokens/sec), and current utilization percentage
- [ ] Time-series data for utilization, job volume, and miner count is available with at least 90-day history and epoch-level granularity
- [ ] Supply vs. demand imbalance signals (e.g., queue depth, average job wait time) are surfaced to flag capacity shortfalls or surpluses
- [ ] Data is broken down by hardware tier to show which tiers are over- or under-utilized
- [ ] An API endpoint exposes the same data in JSON format for governance tooling and analytics integrations

## Notes
Network utilization is a key input to emission rate governance proposals (US-020). If utilization is consistently low, emission rate reductions may be warranted; consistently high utilization signals need for miner recruitment incentives. Make the API endpoint rate-limit-free or minimally restricted for governance transparency.
