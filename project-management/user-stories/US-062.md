# US-062: Preview Estimated Cost Before Submitting an Inference Request

**As a** app developer/API consumer,
**I want to** get a cost estimate for my inference request before committing ROKO tokens,
**So that** I can make informed decisions about prompt length, model selection, and budget allocation before each transaction.

## Acceptance Criteria
- [ ] The SDK `estimateFee(modelId, promptText, maxTokens)` method returns `{ estimatedRoko, estimatedUsd, currentPricePerToken, priceValidUntil }` without submitting any transaction
- [ ] The fee estimate accounts for current market spot price, expected output token count (based on `maxTokens`), and the 5% protocol fee
- [ ] The developer portal provides an interactive fee calculator UI where users can adjust model, input token count, and output token limit to see real-time cost estimates in ROKO and USD
- [ ] Estimates include a confidence interval (e.g., ±10%) reflecting price volatility, and the estimate is flagged as "stale" after the `priceValidUntil` timestamp passes
- [ ] If the estimate is requested for a model or region with no available miners, the response includes a `noMinersAvailable` flag and a suggested alternative model

## Notes
Cost transparency before commitment is a basic UX expectation from developers used to cloud API pricing. The `priceValidUntil` field (criterion 4) is critical — price can change between estimate and submission, and consumers must not be surprised by the actual charge exceeding their estimate by more than the stated confidence interval.
