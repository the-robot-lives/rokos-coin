# US-074: Propose Treasury Allocation for Ecosystem Grants

**As a** Governance Council member,
**I want to** submit and vote on proposals that allocate DAO treasury funds to ecosystem grants, development bounties, or community initiatives,
**So that** the protocol can fund its own growth in a decentralized, accountable manner.

## Acceptance Criteria
- [ ] TreasuryAllocation proposals specify: recipient address(es), token amounts (ROKO or stablecoins held by treasury), vesting schedule (optional), and a human-readable purpose description with IPFS reference
- [ ] Multi-tranche allocations are supported: grant can release in milestones triggered by subsequent governance votes or oracle conditions
- [ ] Treasury contract enforces that no single TreasuryAllocation proposal can exceed 10% of treasury balance without a ProtocolUpgrade-level vote
- [ ] Executed allocations are tracked in a public grants registry on-chain, readable via subgraph
- [ ] Recipients must acknowledge grant terms on-chain (signed acceptance transaction) before first tranche is released

## Notes
Grant registry should include recipient ENS/identity, purpose, milestones, and disbursement history. Consider requiring KYC for large grants (>$50k equivalent) at the discretion of a compliance multisig. Vesting contracts should be audited separately.
