# Web3 Frontend Stack

How to connect a browser application to the blockchain.

## The Current Standard: viem + wagmi

As of 2026, **viem + wagmi** has replaced ethers.js as the default for new projects.

**viem** — A ~27 KB TypeScript library for low-level blockchain interaction. Think of it as the "fetch" for Ethereum.

**wagmi** — React hooks built on top of viem. Handles wallet connection, chain switching, caching, and reactive state.

### Why viem/wagmi Won

| | ethers.js v6 | viem + wagmi |
|---|---|---|
| Bundle size | ~120 KB | ~27 KB (viem core) |
| Tree-shaking | Limited | Excellent |
| Type safety | Basic | Full (infers types from ABIs) |
| React integration | Manual | First-class hooks |
| Active development | Maintained | Rapidly evolving |

**ethers.js v6** is still fine for non-React projects or existing codebases. It's not deprecated — just no longer the default recommendation.

### Basic viem Usage

```typescript
import { createPublicClient, http, parseAbi } from 'viem';
import { mainnet } from 'viem/chains';

const client = createPublicClient({
    chain: mainnet,
    transport: http('https://eth-mainnet.g.alchemy.com/v2/YOUR_KEY'),
});

// Read a contract
const balance = await client.readContract({
    address: '0x...',
    abi: parseAbi(['function balanceOf(address) view returns (uint256)']),
    functionName: 'balanceOf',
    args: ['0x...'],
});

// Get block number
const block = await client.getBlockNumber();
```

### wagmi React Hooks

```typescript
import { useAccount, useBalance, useReadContract, useWriteContract } from 'wagmi';

function TokenBalance() {
    const { address } = useAccount();
    
    const { data: balance } = useReadContract({
        address: '0x...tokenAddress',
        abi: tokenABI,
        functionName: 'balanceOf',
        args: [address],
    });

    const { writeContract } = useWriteContract();

    const handleTransfer = () => {
        writeContract({
            address: '0x...tokenAddress',
            abi: tokenABI,
            functionName: 'transfer',
            args: ['0x...recipient', parseEther('10')],
        });
    };

    return (
        <div>
            <p>Balance: {balance?.toString()}</p>
            <button onClick={handleTransfer}>Send 10 Tokens</button>
        </div>
    );
}
```

### Wallet Connection: RainbowKit

RainbowKit provides a polished wallet connection modal supporting MetaMask, WalletConnect, Coinbase Wallet, and more.

```typescript
import '@rainbow-me/rainbowkit/styles.css';
import { getDefaultConfig, RainbowKitProvider, ConnectButton } from '@rainbow-me/rainbowkit';
import { WagmiProvider } from 'wagmi';
import { mainnet, arbitrum, base } from 'wagmi/chains';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

const config = getDefaultConfig({
    appName: 'Rokos Coin',
    projectId: 'YOUR_WALLETCONNECT_PROJECT_ID',
    chains: [mainnet, arbitrum, base],
});

const queryClient = new QueryClient();

function App() {
    return (
        <WagmiProvider config={config}>
            <QueryClientProvider client={queryClient}>
                <RainbowKitProvider>
                    <ConnectButton />
                    {/* your app */}
                </RainbowKitProvider>
            </QueryClientProvider>
        </WagmiProvider>
    );
}
```

### WalletConnect v2

Not a wallet — a protocol. Enables QR-code and deep-link connections between dApps and mobile wallets. Required for any production dApp that supports mobile users.

You get a `projectId` from cloud.walletconnect.com (free). RainbowKit and wagmi handle the WalletConnect integration automatically.

## ABI: The Contract's API Contract

The ABI (Application Binary Interface) is a JSON file describing a contract's functions, events, and types. It's generated by the compiler and is how your frontend knows how to encode/decode calls.

```json
[
    {
        "type": "function",
        "name": "transfer",
        "inputs": [
            { "name": "to", "type": "address" },
            { "name": "amount", "type": "uint256" }
        ],
        "outputs": [{ "type": "bool" }],
        "stateMutability": "nonpayable"
    },
    {
        "type": "event",
        "name": "Transfer",
        "inputs": [
            { "name": "from", "type": "address", "indexed": true },
            { "name": "to", "type": "address", "indexed": true },
            { "name": "value", "type": "uint256", "indexed": false }
        ]
    }
]
```

**wagmi/viem type inference:** If you pass a `const` ABI (as const assertion), viem infers the exact TypeScript types for function args and return values. This catches bugs at compile time.

```typescript
const abi = [
    {
        type: 'function',
        name: 'balanceOf',
        inputs: [{ name: 'account', type: 'address' }],
        outputs: [{ type: 'uint256' }],
        stateMutability: 'view',
    },
] as const;  // ← the `as const` is critical for type inference
```

## Listening to Events

```typescript
import { useWatchContractEvent } from 'wagmi';

function TransferWatcher() {
    useWatchContractEvent({
        address: '0x...token',
        abi: tokenABI,
        eventName: 'Transfer',
        onLogs(logs) {
            for (const log of logs) {
                console.log(`${log.args.from} → ${log.args.to}: ${log.args.value}`);
            }
        },
    });
}
```

## Transaction Lifecycle in the Frontend

```
User clicks "Send" 
    → writeContract() called
    → Wallet popup appears (MetaMask, etc.)
    → User confirms
    → Transaction submitted to mempool
    → Returns transaction hash immediately
    → Wait for confirmation (useWaitForTransactionReceipt)
    → Transaction mined → UI updates
```

Handle all states: idle → pending wallet → pending chain → confirmed → error.

## Common Frontend Pitfalls

1. **Not handling chain switching** — User might be on the wrong network. Use wagmi's `useSwitchChain`.
2. **BigInt handling** — Token amounts are BigInt (uint256). Don't use `Number()` — it loses precision. Use `formatEther()`, `parseEther()`.
3. **Stale reads** — After a write, cached read data is stale. Use wagmi's `queryClient.invalidateQueries()` or watch for the confirmation event.
4. **Approval UX** — Users hate two-step approve+action. Use ERC-2612 Permit where possible.
5. **RPC rate limits** — Don't poll every second. Use WebSocket subscriptions or event watching.
6. **Wallet disconnection** — Handle gracefully. The user might disconnect mid-flow.
